// Play Console Setup Checklist

## 1. Create Play Console Account
- [ ] Register at https://play.google.com/console
- [ ] Accept Developer Distribution Agreement
- [ ] Pay one-time $25 registration fee

## 2. App Configuration
- [ ] Create new app listing
- [ ] Set package name: [your.package.name]
- [ ] Add store listing:
  - [ ] App title (≤30 chars): "Kojori Bus"
  - [ ] Short description (< 80 chars)
  - [ ] Full description
  - [ ] Screenshots (minimum 2, recommended 5+)
  - [ ] Feature graphic
  - [ ] Promo video (optional, disable ads if used)
  - [ ] Content rating questionnaire

## 3. Legal & Content Requirements
- [ ] Add Privacy Policy URL in Store Listing
- [ ] Add your website URL (required for contact)
- [ ] Declare app content classification
- [ ] Complete Data Safety form:
  - [ ] Data collected (list all)
  - [ ] Data shared (if any)
  - [ ] Data security practices
- [ ] If using location: declare in Data Safety

## 4. Target API Levels
- [ ] Target Android 14 (API 34) for new apps
- [ ] Test on Android 13+ minimum
- [ ] Update compileSdkVersion to 34

## 5. Required Declarations
- [ ] App content type declaration
- [ ] Age-based content ratings
- [ ] If collecting user data: Data Safety form
- [ ] If contains ads: declare ad networks
- [ ] If offers financial/health services: Organization registration

## 6. Build Configuration
- [ ] Update signing configuration
- [ ] Generate signed APK/AAB
- [ ] Version code: increment (e.g., 2)
- [ ] Version name: update (e.g., 1.1.0)
- [ ] Verify proguard/R8 settings

## 7. Pre-Submission Testing
- [ ] Test on physical Android device
- [ ] Verify offline functionality works
- [ ] Test both directions (Kojori ↔ Tbilisi)
- [ ] Verify widget rebuild works
- [ ] Check location permissions flow
- [ ] Test background data usage

## 8. Submission
- [ ] Upload APK/AAB
- [ ] Complete content rating
- [ ] Submit for review
- [ ] Monitor review status
- [ ] Address any rejections

## 9. Post-Submission
- [ ] Track review progress
- [ ] Respond to reviewer questions
- [ ] Update app as needed based on feedback

## Key Compliance Points for Transit Apps:
- ✓ Location permission is optional (don't block core functionality)
- ✓ Privacy policy explains data usage
- ✓ No misleading claims about accuracy
- ✓ Age rating appropriate for general audience
- ✓ Data Safety form completed accurately
- ✓ Widget rebuild instructions documented

## Common Rejection Reasons to Avoid:
- Missing privacy policy
- Incomplete Data Safety form
- Misleading store descriptions
- Unnecessary permission requests
- Targeting children without proper certification
- Non-compliant content/advertising
